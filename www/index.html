01

HTTP server nedir?

HTTP sunucusu (veya web sunucusu), internet tarayıcılarından gelen istekleri HTTP protokolü üzerinden karşılayan, içerikleri saklayan ve ileten yazılım ve donanımlardır. Bu proje ise bu sunucunun yazılımsal kısmını ifade eder.

İstemci sunucuya metin tabanlı bir HTTP isteği yollar. Sunucu bu metni yorumlar, uygun cevabı hazırlar ve aynı TCP bağlantısı üzerinden geri döndürür.

İlgili dosyalar: README.md src/main.c, src/server.c

04

Socket kurulumu (server.c:main fonksiyon içinde)

Sunucunun internetten gelen istekleri dinleyebilmesi için ilk olarak bir boş bir soket açmak gerekir.



struct sockaddr_in server_addr; // Sunucu ayarlarını tutan hazır struct
int server_fd; // Sunucu için bir file descriptor
// Server socketi oluşturma AF_INET: IPv4 SOCK_STREAM: akış tabanlı, 0: protokol: varsayılan protokolü seç
if ((server_fd = socket(AF_INET, SOCK_STREAM, 0)) < 0)
{
    exit(EXIT_FAILURE);
}

Sunucu ayarlarını yapmak için sunucu struct kirli veriden temizlenir


memset(&server_addr, 0, sizeof(server_addr));
server_addr.sin_family = AF_INET;
server_addr.sin_addr.s_addr = INADDR_ANY; // Makinenin tüm yerel IPv4 arayüzlerinden gelen bağlantıları dinle.
server_addr.sin_port = htons(PORT);

bind() ile struct içindeki ayarlar sokete uygulanır.


// Socket'i server_fd'ye bağlama işlemi
if (bind(server_fd, (struct sockaddr *)&server_addr, sizeof(server_addr)) < 0)
{
    exit(EXIT_FAILURE);
}

listen() komutu ile soket istekleri dinlemeye başlar


// BACKLOG : kabul edilmeyi bekleyen bağlantı kuyruğunun boyutu
if (listen(server_fd, BACKLOG) < 0)
{
    exit(EXIT_FAILURE);
}

05

Thread havuzu ve thread safe kuyruk

Her bağlantı için yeni thread açmak yerine sabit sayıda worker thread kullanılır. Ana thread client fd değerlerini thread-safe kuyruğa koyar. 


pthread_mutex_t queue_mutex = PTHREAD_MUTEX_INITIALIZER; //Kuyruğu trade safe yapmak için
pthread_cond_t queue_cond = PTHREAD_COND_INITIALIZER; //Thread durum yönetimi için
pthread_t threads[THREAD_COUNT]; //Threadleri yönetmek için dizi

int queue[QUEUE_SIZE]; // Client_fd 'leri tutan trade safe kullandığımız dizi
int front = 0;
int rear = 0;
int count = 0;

Threadlerin oluşturulması (server.c:main fonksiyon içinde)

                       
for (int i = 0; i < THREAD_COUNT; i++)
{
    pthread_create(&threads[i], NULL, worker_thread, NULL);
}

Worker thread'ler kuyruktan iş alır, request'i işler ve bağlantıyı kapatır. Böylece aynı anda birden fazla istek işlenebilir ama thread sayısı kontrol altında kalır.

Threadin yapacağı komutların fonksiyonu


void *worker_thread()
{
    while (1)
    {
        int client_fd = queue_pop(); //Kuyruktan iş çekmeye çalışır.

        if (client_fd == -1)
        {
            break; // Döngüden çık ve thread'i temizce sonlandır.
        }

        _handle_client(client_fd);
        close(client_fd);
    }

    return NULL;
}

06

HTTP isteği (request) okuma ve yorumlanması

Client_fd 'yi kuyruktan alan thread _handle_client() fonksiyonu ile gelen isteği işlemeye başlar. İsteği işleyebilmek için HTTP protokolünü ve gelecek mesajın şablonunu bilmek gerekir.

HTTP request 4 ana parçadan oluşur: request line, header, boşluk ve body.

Örnek bir HTTP POST isteği


POST /api/ogrenci HTTP/1.1\r\n
Host: example.com\r\n
Content-Type: application/json\r\n
Content-Length: 64\r\n
Authorization: Bearer TOKEN_BURAYA_YAZILIR\r\n
Accept: application/json\r\n
\r\n
{
    "adSoyad": "Ali Veli",
    "ogrenciNo": "123456",
    "bolum": "Bilgisayar Mühendisliği"
}

İlk satırdan method, path ve HTTP version bilgisini içerir. Altından gelen satırlar ise gelen istek hakkındaki detaylı bilgilerdir.

\r\n\r\n karakter örüntüsüne kadar gelen bütün satırlar header içerisine dahildir.

Header ve body arasındaki ayrım \r\n\r\n karakter dizisi ile tespit edilir.

Ayrımdan sonra gelen bütün ifade body kısmıdır.

Artık kullanıcıdan gelen mesajın neye benzeyeceğini bildiğimize göre mesajı çekmeye başlayabiliriz.

_handle_client() fonksiyonunda ilk olarak gerekli değişkenler tanımlanır.


static void *_handle_client(int client_fd)
{
    int _toutErrorCode = 0; // İstek zaman aşımı bayrağı
    char tempBuffer[MAX_REQ_SIZE + 1] = {0}; // Kullanıcıdan gelen ham verinin doldurulacağı buffer
    size_t total_bytes = 0; // (t) anına kadar yakalanmış toplam byte

    char *endOfHeader = NULL;  // Header sonunu işaret edecek gösterici (pointer)
    char *startBody = NULL; // Body başlangıcını işaret edecek gösterici (pointer)
    size_t header_size;  // Header boyutunu tutan değişken
    ...}

_read_header() fonksiyonu ile isteğin header/body ayrımını yapan örüntü yakalanmaya çalışılır.


endOfHeader = _read_header(client_fd, tempBuffer, &total_bytes, &_toutErrorCode);

Eğer endOfHeader için bir değer dönülmüşse header_size = endOfHeader - tempBuffer ile header boyutu hesaplanır.

Header boyutu tanımlanan maksimum boyutunu aşmadıysa isteğin yorumlanması için parçalanması gerekir. Parçalama işlemi için ayrı bir buffer ve parçalanan isteğe ait bilgilerin tutulacağı struct yapı olan HttpRequest değişkeni oluşturulur. 


char headerBuffer[header_size + 1];
strncpy(headerBuffer, tempBuffer, header_size);
headerBuffer[header_size] = '\0';

HttpRequest httpRequest = {0};

headerBuffer ve httpRequest parse_http_request_header() fonksiyonuna gönderilir.


int parse_http_request_header(char *buffer, HttpRequest *httpRequest)
{
    char *current_pos = buffer; // Header başlangıcını tutan pointer
    char *next_line = strstr(current_pos, "\r\n"); //İlk satırın sonunu gösteren pointer 
    ...
}

Eğer next_line tespit edilirse size_t status_line_length = next_line - current_pos; ile ilk satırın boyutu hesaplanır. Metod, yol ve HTTP versiyonu verilerini çekmek için geçici ilk satır dizisi oluşturulur. 


char tempStatusLine[status_line_length + 1];
strncpy(tempStatusLine, current_pos, status_line_length);
tempStatusLine[status_line_length] = '\0';

sscanf() metodu ile ilk satırdan verileri çekilir ve HttpRequest struct'ı içine kayıt edilir.


int parsed_count = sscanf(tempStatusLine, "%15s %255s %15s", httpRequest->method,
httpRequest->path, httpRequest->version);

Body içeren HTTP metotlarında body boyutu Content-Length: # olarak header içinde gönderilir. Bu bilgi mühimdir çünkü gönderilen body eksiksiz olması gerekir. Eğer eksik ise Content-Length kadar byte okunana kadar soketten okumaya devam edilir.

parse_http_request_header() içinde _find_content_length() ile eğer varsa Content-Length: bulunmaya çalışır.

Bu projede tasarım tercihi olarak GET isteğinde body varsa istek reddedilir.

parse_http_request_header() metodu hatasız tamamlanırsa return 0; döner.

_handle_client() fonksiyonu içinde bir sonraki adım eğer varsa body içeriğinin eksiksiz olarak elde edilmesidir.


size_t current_body_size = total_bytes - (header_size + 4);
if (current_body_size > 0)
    {
        startBody = endOfHeader + 4; // \r\n\r\n atlanarak body'nin başlangıcı bulunur
    }

Eğer current_body_size beklenen Content-Length'ten az ise tekrar soket üzerinden tekrar veri okuması yapılır. Body boyutunun izin verilen maksimum kapasiteden az olmasına dikkat edilmesi gerekir. Edilmemesi zafiyet oluşturur.

HTTP Request işlenip struct elemanı içinde tanımlamaları yapıldıktan sonra route_request() fonksiyonuna yönlendirilir.

07

Router sistemi

Router, gelen request'in method ve path bilgisine bakarak hangi handler'in çalışacağına karar verir. GET / statik dosyaya, POST /api/hello API handler'ina gider.

Bu katman projenin trafik polisi gibidir: request doğru yola buradan yönlendirilir.


int route_request(int client_fd, HttpRequest *httpRequest)
{

    if (strcmp(httpRequest->method, "GET") == 0)
    {
        if (strcmp(httpRequest->path, "/") == 0)
        {

            return sendStaticFile(client_fd, "/index.html");
        }

        return sendStaticFile(client_fd, httpRequest->path);
    }
    else if (strcmp(httpRequest->method, "POST") == 0)
    {
        char *is_api = strstr(httpRequest->path, "/api/");

        if (is_api == NULL)
        {
            LOG_ERROR("Bad path: %s\n", httpRequest->path);
            send_HttpError_Response(client_fd, 400); // bad request

            return -1;
        }

        if (is_api == httpRequest->path) // /api/ ilk başta mı
        {
            return handleApi(client_fd, httpRequest);
        }
        else
        {
            send_HttpError_Response(client_fd, 400); // bad request
            return -1;
        }
    }
    else
    {
        send_HttpError_Response(client_fd, 400); // bad request
        return -1;
    }

08

Statik dosya sunumu

Statik dosya sunumu, URL path'inin güvenlik kontrolü ile başlar. .., \ ve % karakterleri reddedilir.

Not: Yüzde işaretinin kabul edilmemesi tasarım basitliği tercihinden dolayıdır.

URL güvenli ise statik dosyaların bulunduğu dizin olan ./www ile string düzeyinde birleştirilerek işletim sisteminin anlayacağı dosya yolu elde edilir. 


char *fileRoot = "./www";
size_t filePathSize = strlen(fileRoot) + strlen(path) + 1;
char *filePath = (char *)calloc(filePathSize, 1);

Dosya yolu fopen() sistem çağrısı ile açılmaya çalışılır.


FILE *file = fopen(filePath, "rb");

Eğer dosya mevcutsa Http response struct'ı tanımlanarak response oluşturulmaya başlanır.

Response header'ına gönderilecek dosya formatının bilgisini içeren MIME tipi bilgisi setMIMEtype() fonksiyonu ile atanır. prepfileSize() fonksiyonu ile dosya boyutu hesaplanır ve response yapısına atanır. 


HttpResponse httpResponse;
httpResponse.MIMEtype = setMIMEtype(filePath);
if (prepfileSize(file, &httpResponse) != 0)
    {
        send_HttpError_Response(client_fd, 500);
        goto cleanup;
    }

Not: prepfileSize() fonksiyonu başarısız olursa Sunucu içi hata kodu gönderilir.

Gönderilecek dosyanın boyutu ve türü tespit edildikten sonra prepHttpResponseHeader() fonksiyonu ile response header string olarak hazırlanır ardından sendHttpResponseHeader() fonksiyonu ile istemciye yollanmaya çalışılır.

Header gönderildikten sonra body yani asıl içerik olan statik dosya sendFileContent() fonksiyonu ile gönderilmeye çalışılır.

Not: Response gönderim işleminin detayları açıklanmıştır.

http.c dosyası ve veri gönderme detayları

Projede bulunan http.c dosyası, HTTP hata mesajlarını, statik dosya ve api cevaplarını sistem çağrıları kullanarak gönderen fonksiyonları içerir.

Hata mesajlarının içerikleri güncellenebileceği için hata mesajlarının header ve tüm response metinleri dinamik olarak init... fonksiyonları ile sunucu başlatılırken oluşturulur.

Dosya ve API cevapları da HttpResponse yapısı (struct) içindeki veriler kullanılarak oluşturulurlar.


// Response Header dinamik olarak hazırlanır.
int prepHttpResponseHeader(HttpResponse *httpResponse)
{
    strcpy(httpResponse->statusLine, httppOKStatus);
    size_t size = sizeof(httpResponse->headers);
    int n = snprintf(httpResponse->headers, size,
                     "Content-Type: %s\r\n"
                     "Content-Length: %ld\r\n"
                     "Connection: close\r\n"
                     "Cache-Control: no-store\r\n"
                     "\r\n",
                     httpResponse->MIMEtype,
                     httpResponse->fileSize);

    if (n < 0 || n >= (int)size)
    {
        LOG_ERROR("Creating response header didnt work.\n");
        return -1;
    }

    return 0;
}

// Api hatalarının Response Headeri dinamik olarak hazırlanır.
int prepApiErrorHeader(HttpResponse *httpResponse)
{
    strcpy(httpResponse->statusLine, httpBadRequestStatus);
    size_t size = sizeof(httpResponse->headers);
    int n = snprintf(httpResponse->headers, size,
                     "Content-Type: %s\r\n"
                     "Content-Length: %ld\r\n"
                     "Connection: close\r\n"
                     "Cache-Control: no-store\r\n"
                     "\r\n",
                     httpResponse->MIMEtype,
                     httpResponse->fileSize);

    if (n < 0 || n >= (int)size)
    {
        LOG_ERROR("Creating response header didnt work.\n");
        return -1;
    }

    return 0;
}

Response gönderimi sırasında çeşitli sebeplerden dolayı veriler tek seferde gitmeyebilir. Mesajın gitmesinden emin olmak için _sendAll() fonksiyonu oluşturulmuştur. Bütün response çeşitleri (statik, api vb.) bu fonksiyon ile iletilerini yollarlar. 


 int sendFileContent(int client_fd, FILE *file)
{
    char BUFFER[BUFFER_SIZE];
    size_t bytesRead;

    while ((bytesRead = fread(BUFFER, 1, BUFFER_SIZE, file)) > 0)
    {
        _sendAll(client_fd, BUFFER, bytesRead);
    }
    return 0;
}

int sendApiContent(int client_fd, char *json_string)
{
    size_t size = strlen(json_string);

    _sendAll(client_fd, json_string, size);

    return 0;
}

_sendAll() fonksiyonu gönderilmek istenen byte sayısına ulaşılana kadar send() yerleşik C fonksiyonu ile alıcıya byte gönderir. Her gönderim sırasında başarıyla giden byte'lar toplanarak gönderilmek istenen miktarla karşılaştırılır. bytesSent == bytesRead olduğu anda ise gönderim tamamlanmıştır. 


 int _sendAll(int client_fd, const char *buffer, size_t bytesRead)
{
    size_t bytesSent = 0;
    size_t bytesRemaining;
    ssize_t n;
    const char *bufferStartPoint;

    while (bytesSent < bytesRead)
    {
        bufferStartPoint = buffer + bytesSent;
        bytesRemaining = bytesRead - bytesSent;
        n = send(client_fd, bufferStartPoint, bytesRemaining, 0);

        if (n <= 0)
        {
            LOG_ERROR("send() function: -1.\n");
            return -1;
        }

        bytesSent += n;
    }

    return 0;
}
